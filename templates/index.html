Platform Highlights

The goal of this first page is to introduce the supported scope of the project before the user enters the upload and prediction workflow.

01

Multi-plant support

The project distinguishes among potato, rose, and tomato leaves instead of forcing every image into a single plant category.

02

Disease-aware prediction

Each plant has disease classes trained separately so the output includes both the plant type and the disease label.

03

Dedicated detector page

The upload workflow is separated from the landing page, making the project cleaner, easier to present, and simpler to navigate.

How It Works

How the platform works

The system follows a clear sequence from image upload to prediction output, which makes it easier to explain in a presentation.

Step 1

Upload the image

The user uploads a leaf image from one of the supported plants on the detection page.

Step 2

Preprocess and classify

The backend resizes and normalizes the image before sending it to the selected deep learning model.

Step 3

Show diagnosis

The frontend displays the plant name, disease name, model used, and confidence score in a structured result panel.
